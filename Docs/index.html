如何使用

ICECSS是一个以冰山为灵感的开源高效的基于JQuery的CSS框架，不仅有着开发快速、美观易用等特点，官方本身还提供CDN服务、快速开发demo和测试的云服务等等。ICECSS支持通过less引用也支持普通方式

您可以通过直接下载的方式获取，也可以通过bower：

bower install icecss

或者

bower install besd-icecss

--普通方式调用--

在html文件头部引用ICECSS的CSS

 <link rel="stylesheet" href="css/icecss.css>

在html文件尾部引用ICECSS的JS（注意，ICECSS是基于JQuery的，所以在引用ICECSS的相关JS文件前请先导入Jquery）

 <script src="js/jquery.min.js"></script>
 <script src="js/icecss.js"></script>

--Less调用--

 <link rel="stylesheet/less" href="css/icecss.less>

在html文件尾部多引用一个less.min.js即可。更多有关Less用法

你也可以加入我们的交流群：320283384

感谢很多的开源项目，如jquery、normalize.css等等，因为有了你们才有ICECSS。另外有些插件是我们基于一些开源项目封装的，一般都有注释封装自哪个项目。如果您认为的项目在ICECSS中侵犯了您的权益，可以与我们联系：t-baby@besdlab.cn。

如果您想要赞助ICECSS的研发，也可以通过邮件与我们联系。

基础元素

所有的基础元素如h1等，我们要求必须在ice-div下才会生效。

同理，ice-div下基础元素都不需要额外的增加class。

如：

<div class="ice-div"><h1>TEST</h1></div>

基础元素包含：

  • h1
  • h2
  • h3
  • h4
  • h5
  • h6
  • p
  • ul
  • li
  • ol
  • dl
  • dt
  • dd
  • a
  • blockquote
  • hr（分割线）

如果你需要有引用标题的效果，可以这么做：

<p class="ice-title">test</p>

面板：

您可以使用ice-panel这个class来建立一个面板，这个是默认的样式，是灰框白底。

不需要任何边框和背景的话，可以使用ice-panel-original这个class。

ICECSS专门为您提供了用于提示的ice-panel-point面板。 还提供了用于关闭父元素的ice-close，您可以将这个用于关闭提示面板。

<div class="ice-panel-point">
    <span class="ice-close"></span>
    <p>这是一个可关闭的提示面板</p>
</div>

预置颜色

我们在ICECSS的Less中预置了一些颜色。

 @color-common: #297fb8;
 @color-white: #FFFFFF;
 @color-blue-1: #3598DC;
 @color-blue-2: #2A80B9;
 @color-green-1: #1bbc9d;
 @color-green-2: #16A086;
 @color-green-3: #2FCC71;
 @color-green-4: #27AE61;
 @color-yellow-1: #F1C40F;
 @color-yellow-2: #F49C14;
 @color-orange-1: #E77E23;
 @color-orange-2: #D55401;
 @color-red-1: #E84C3D;
 @color-red-2: #C1392B;
 @color-purple-1: #9C59B8;
 @color-purple-2: #8F44AD;

而主体颜色都是通过引用@color-common这个变量的，如果需要改变主题颜色可以直接将颜色代码替换到变量后面。

如我要替换成黄色：@color-common: #F1C40F;

当然你也可以直接通过CSS来使用，我们在CSS中提前准了一些类，可以通过ice-color-颜色名来使用。如果是边框请用ice-color-b+颜色名。

如.ice-color-blue-1和.ice-color-bblue-1

布局

ICECSS的布局会自动适应屏幕。但是请不要把导航栏放在ICECSS的布局中，因为ICECSS的导航栏有自己独有的自适应能力。

我们为您提供了三种宽度限制：ice-common、ice-widescreen、ice-full 。分别宽度限制是1020px、1220px和100%。 当然您也可以不增加宽度限制，直接使用ice-div。

ICECSS的DIV组件提供如下几种class，第一个数字为分子，第二个数字为分母。每排可以容纳100%。

  • ice-div-1-1
  • ice-div-1-2
  • ice-div-1-3
  • ice-div-1-4
  • ice-div-1-6
  • ice-div-4-10、ice-div-6-10
  • ice-div-7-10、ice-div-3-10
  • ice-div-2-10、ice-div-8-10
  • ice-div-9-10、ice-div-1-10

你需要在每个class前加入ice-div才能使用，如 :

<div class="ice-div ice-div-1-1"></div>

并且你需要在每行的最后面的div那里加入ice-div-last，来换行。如:

<div class="ice-div ice-div-1-1 ice-div-last"></div>

在ice-div下图片也是会自动适应的，不需要增加额外的class。

水平居中请使用.ice-center-plane；绝对居中的话在父元素使用 .ice-center，子元素使用.ice-center-abs 。

导航栏

（1）菜单简介

在ICECSS中，导航栏是Menu组件。可以自适应屏幕。我们建议在header标签中放置Menu组件，建议不要放在ICECSS的布局组件中。

（2）基本菜单 例子：

  <header class="ice-menu">
<nav class="ice-main-nav">
<ul>
<li><a href="#0">Home</a>
</li>
<li><a href="#0">Home</a>
</li>
</ul>
</nav>
</header>

（3）在导航栏左侧增加LOGO：

<div class="ice-menu-logo">LOGO名字</div>

把代码增加到header下面的第一行。 使导航栏固定： 在ice-menu后面追加 ice-fixed

<header class="ice-menu ice-fixed">

3.使导航栏透明，去掉背景色： 在ice-menu后追加ice-menu-lucency

 <header class="ice-menu ice-fixed ice-menu-lucency">

4.使导航栏在鼠标悬浮时有效果： 追加ice-menu-hover

<header class="ice-menu ice-fixed ice-menu-lucency ice-menu-hover">

（4）全屏大图 我们在菜单组件中额外的增加了一个全屏大图用的组件ice-intro-big，可以搭配其它组件一起使用。

提示：为了达到最好的效果，不建议放在ICECSS布局系统的宽度限制下。 使用例子：

<div class="ice-div ice-intro-big ice-center" style="background-image:url('img/bg.jpg')">
<div class="ice-center-abs">
<h1 style="color:#fff">ICECSS is a good css.</h1>

</div> </div>

代码高亮

使用前需要引入plugin/ice_code下的文件，默认是支持html、css、js，如果需要更多语言请导入ice_code_expands.js，导入扩展包后增加了对c、go、php、ruby、java、markdown、git的代码高亮支持。

<pre><code class="language-这里填语言">这里面放置代码</code></pre>

中文优化

ICECSS针对中文适配了优化方案，只需要在需要优化显示的地方加上ice-chinese这个class。

<div class="ice-chinese"></div>

云服务

BESD云服务是用于帮助快速开发DEMO和测试用的云服务平台，ICECSS0.8版本后已经附带了JS版的SDK。通过BESD云服务可以在不写后端代码的情况下快速完成DEMO或简单程序的开发。如果您是要在ICECSS中使用，请将将plugin/ice_cloud.js引入

说明文档：https://github.com/T-baby/BesdCloud

模板引擎

ICECSS自带一个简单的模板引擎，可以使用户界面与业务数据（内容）分离，从而在前端开发出动态渲染的网站。同时可以更好的帮助前端处理json数据。如果您是要在ICECSS中使用，请将将plugin/ice_tmpl/ice_tmpl.js引入

文档地址：https://github.com/T-baby/ICECSS/tree/master/Docs/ice_tmpl/readme.md

平滑移动

在需要平滑移动效果的超链接的父类那里增加sll,然后在script里增加：

 $(".sll").ice_scroll(700)

滚动监控

在script中使用这个命令来监控：

 $('监控对象').waypoint(function() {     });

可以使用下面的代码进行设置：

$.fn.waypoint.defaults = {
context: window,
continuous: true,
enabled: true,
horizontal: false,
offset: 100,
triggerOnce: false
}

图片延迟加载

图片延迟加载可以让网页在载入的时候不载入图片，当看到图片所在的部分的时候再载入。使用前需要引入plugin/ice_image.js。图片按照下面的方式加入到网页中，ajax.gif是等待加载的动画，后面的属性是需要加载的图片

<img src="../img/ajax.gif" alt="" data-echo="../img/album-1.jpg">

然后在js中添加方法

ice_echo.init()

超链接预载

超链接预载可以在鼠标悬浮在超链接上时提前加载网页，可以减少网页加载的等待时间。使用前需要引入plugin/ice_click.js。如果在导入脚本的script标签添加data-no-instant属性的话，超链接预载只会使用一次，去掉的话则每次都会使用。

<script src="ice_click.js" data-no-instant></script>

然后在js中添加方法

ice_click.init();

本地存储

ice_storedb是一个基于localStorage的本地储存库，通过模拟MongoDB的一些API和概念（如“集(collection)”和“文档(document)”），使你能使用 localStorage 储存复杂数据。使用前需要引入plugin/ice_storedb.js。

插入（Insert）

向名为players的集合中插入一条文档：


ice_storedb('players').insert({"name":"Randy","sex":"male","score":20},function(err,result){
  if(!err){
    //do sth...
  } else{} //do sth...
})

查询（Find）

查询players集合中name为Randy的文档：


ice_storedb('players').find({"name":"Randy"},function(err,result){
  if(!err){
    //use result to do sth...
  } else{} //do sth...
})

如果需要查询集合中所有文档，将参数设置为空即可：


ice_storedb('players').find()

函数将返回一个数组类型。

更新（Update）

为players集合中name为Randy的score增加10：


ice_storedb('players').update({"name":"Randy"},{"$inc":{"score":"10"}},function(err){
  if(!err){
    //do sth...
  } else{} //do sth...
})

你可能已经注意到，ice_StoreDB拥有和MongoDB一样的修改器！关于修改器类型请查看API。

如果修改器为空，则默认为$set修改器：


ice_storedb('players').update({"name":"Randy"}, {"sex":"male","name":"kriss"})

删除（Remove）

删除在players集合中name为Randy的一条文档：


ice_storedb('players').remove({"name":"Randy"},function(err){
  if(!err){
    //do sth...
  } else{} //do sth...
})

如果要把整个集合删除，把参数设置为空：


ice_storedb('players').remove()
